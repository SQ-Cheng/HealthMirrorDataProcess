# `mirrorx_data` 文件夹数据说明

`mirrorx_data`（如 `mirror1_data/`, `mirror2_data/` 等）用于存放多模态生理特征提取及相关评估任务的原始采集和初步处理数据。数据按采集对象（Patient）进行划分，每个文件夹代表一个数据镜像源设备或收集节点。

## 1. 总体目录树结构

一个典型的 `mirrorx_data` 文件夹的结构如下：

```text
mirrorX_data/
└── patient_000116/                   # 以具体的患者/受试者 ID 命名的子文件夹
    ├── patient_info.txt              # 患者及采集相关的基础信息
    ├── video.avi                     # RGB 光源视频
    ├── video.avi.ts                  # RGB 视频的时间戳文件
    ├── ir_video.avi                  # 红外（IR）光源视频
    ├── ir_video.avi.ts               # 红外视频的时间戳文件
    ├── images/                       # 从 RGB 视频提取出的图像序列文件夹
    │   └── timestamps.txt            # 该文件夹下图像的时间戳信息
    ├── ir_images/                    # 从红外视频提取出的图像序列文件夹
    │   └── timestamps.txt            # 该文件夹下红外图像的时间戳信息
    ├── ecg_log.csv                   # 原始心电 (ECG) 信号数据的日志 
    ├── rppg_log.csv                  # 评估提取出的 rPPG (远程光电容积脉搏波) 信号 
    ├── merged_log.csv                # 时间戳对齐合并后的多个生理信号
    ├── normalized_log.csv            # 经过滤波和归一化处理后的生理数据
    └── log.csv                       # 推理和系统运行日志
```

## 2. 核心数据文件解析

### 2.1 基础信息
* **`patient_info.txt`**：
  * 保存该病例的基础信息。
  * **内容格式**：包含 `Patient ID`、采集的开始时间 `Session Timestamp` 以及包含如设备等详细信息的 JSON 字符串格式化内容。

### 2.2 视频和影像流数据
* **`video.avi` & `ir_video.avi`**：
  * 分别存储正常可见光 (RGB) 与红外光 (IR) 两种模态的面部/皮肤录像。
* **`*.avi.ts` (时间戳序列文件)**：
  * 对应每一个具体视频帧的Unix时间戳记录，用于后续多模态和生理信号（如心电）跨设备对齐。

### 2.3 生理信号与波形记录 (CSV 文件)
所有的波形和信号记录文件均带有相对应的时间戳，用于时序分析：
* **`ecg_log.csv`**：
  * **字段定义**：`时间戳信息, ECG信号幅值`。
  * **作用**：真实接触式传感器获取的 Ground Truth（金标准），用来作为评估 rPPG 的基准。
* **`rppg_log.csv`**：**不建议使用, 请看下一部分**
* **`merged_log.csv`, `normalized_log.csv`**：**不建议使用**

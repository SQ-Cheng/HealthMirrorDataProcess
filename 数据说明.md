# `mirrorx_data` 文件夹数据说明

`mirrorx_data`（如 `mirror1_data/`, `mirror2_data/` 等）用于存放多模态生理特征提取及相关评估任务的原始采集和初步处理数据。数据按采集对象（Patient）进行划分，每个文件夹代表一个数据镜像源设备或收集节点。

## 1. 总体目录树结构

一个典型的 `mirrorx_data` 文件夹的结构如下：

```text
mirrorX_data/
└── patient_000116/                   # 以具体的患者/受试者 ID 命名的子文件夹
    ├── patient_info.txt              # 患者及采集相关的基础信息
    ├── video.avi                     # RGB 光源视频
    ├── video.avi.ts                  # RGB 视频的时间戳文件
    ├── ir_video.avi                  # 红外（IR）光源视频
    ├── ir_video.avi.ts               # 红外视频的时间戳文件
    ├── images/                       # 从 RGB 视频提取出的图像序列文件夹
    │   └── timestamps.txt            # 该文件夹下图像的时间戳信息
    ├── ir_images/                    # 从红外视频提取出的图像序列文件夹
    │   └── timestamps.txt            # 该文件夹下红外图像的时间戳信息
    ├── ecg_log.csv                   # 原始心电 (ECG) 信号数据的日志 
    ├── rppg_log.csv                  # 评估提取出的 rPPG (远程光电容积脉搏波) 信号 
    ├── merged_log.csv                # 时间戳对齐合并后的多个生理信号
    ├── normalized_log.csv            # 经过滤波和归一化处理后的生理数据
    └── log.csv                       # 推理和系统运行日志
```

## 2. 核心数据文件解析

### 2.1 基础信息
* **`patient_info.txt`**：
  * 保存该病例的基础信息。
  * **内容格式**：包含 `Patient ID`、采集的开始时间 `Session Timestamp` 以及包含如设备等详细信息的 JSON 字符串格式化内容。

### 2.2 视频和影像流数据
* **`video.avi` & `ir_video.avi`**：
  * 分别存储正常可见光 (RGB) 与红外光 (IR) 两种模态的面部/皮肤录像。
* **`*.avi.ts` (时间戳序列文件)**：
  * 对应每一个具体视频帧的Unix时间戳记录，用于后续多模态和生理信号（如心电）跨设备对齐。

### 2.3 生理信号与波形记录 (CSV 文件)
所有的波形和信号记录文件均带有相对应的时间戳，用于时序分析：
* **`ecg_log.csv`**：
  * **字段定义**：`时间戳信息, ECG信号幅值`。
  * **作用**：真实接触式传感器获取的 Ground Truth（金标准），用来作为评估 rPPG 的基准。
* **`rppg_log.csv`**：**不建议使用, 请看下一部分**
* **`merged_log.csv`, `normalized_log.csv`**：**不建议使用**

## 3. 患者信息汇总文件

通过 `main_patient.py` 的子命令可以从各 mirror 数据目录中提取并合并患者信息，生成 CSV 汇总文件。

### 3.1 `merged_patient_info_X.csv`

由 `main_patient.py extract` + `main_patient.py merge` 生成（对应脚本 `extract_patient_info.py` 和 `merge_patient_info.py`）。每行对应一次采集记录（同一患者可能有多行）。

| 列名 | 类型 | 说明 |
|------|------|------|
| `lab_patient_id` | int | 采集系统分配的顺序 ID（如 mirror 6 中 1–166） |
| `hospital_patient_id` | int | 医院病案号，用于关联医院系统数据；缺失时为 `-1` |
| `blood_oxygen` | int | 血氧饱和度（%），缺失时为 `-1` |
| `heart_rate` | int | 心率（bpm），缺失时为 `-1` |
| `respiratory_rate` | int | 呼吸频率（bpm），缺失时为 `-1` |
| `temperature` | float | 体温（°C），缺失时为 `-1` |
| `low_blood_pressure` | int | 舒张压 / 低压（mmHg），缺失时为 `-1` |
| `high_blood_pressure` | int | 收缩压 / 高压（mmHg），缺失时为 `-1` |

### 3.2 `merged_patient_info_with_lab_X.csv`

由 `main_patient.py merge-lab` 生成（对应脚本 `merge_lab_xlsx.py`）。在 `merged_patient_info_X.csv` 的基础上，通过 `hospital_patient_id` 左连接 `健康镜化验1.0.xlsx` 中的化验数据，追加以下列。未匹配到的行以 `-1` 填充。

**数据来源**：`健康镜化验1.0.xlsx` 包含 329 名患者的 5154 条乳酸浓度检验记录（来自 10 个检验套），每名患者有多次测量。合并时按患者聚合为单行统计量。

| 列名 | 类型 | 说明 |
|------|------|------|
| `gender` | str | 性别（`男` / `女`），未匹配时为 `-1` |
| `age` | float | 就诊时年龄（岁），未匹配时为 `-1` |
| `lactate_min` | float | 乳酸浓度最小值（mmol/L），保留 2 位小数 |
| `lactate_max` | float | 乳酸浓度最大值（mmol/L），保留 2 位小数 |
| `lactate_mean` | float | 乳酸浓度均值（mmol/L），保留 2 位小数 |
| `lactate_median` | float | 乳酸浓度中位数（mmol/L），保留 2 位小数 |
| `lactate_std` | float | 乳酸浓度标准差（mmol/L），保留 2 位小数；仅 1 次测量时为 `NaN` |
| `lactate_count` | float | 乳酸测量次数 |
| `lactate_first_report_time` | str | 最早报告时间（`YYYY-MM-DD HH:MM:SS`），未匹配时为 `-1` |
| `lactate_last_report_time` | str | 最晚报告时间（`YYYY-MM-DD HH:MM:SS`），未匹配时为 `-1` |

**各镜像匹配情况**（截至 2026-05-06）：

| 镜像 | 源 CSV 行数 | 匹配到化验数据 | 未匹配 |
|------|------------|---------------|--------|
| mirror 1 | 523 | 89 | 434 |
| mirror 2 | 347 | 323 | 24 |
| mirror 4 | 256 | 115 | 141 |
| mirror 5 | 315 | 146 | 169 |
| mirror 6 | 166 | 141 | 25 |

> **注意**：`merged_patient_info_with_lab_X.csv` 是独立的新文件，不会覆盖原始的 `merged_patient_info_X.csv`。
